## Day 1：V1（稀疏奖励）训练

### Step 1.1 — 拉最新 V1 代码（5 分钟）

```bash
cd ~/project/hilserl-surrol-improved   # 或你本地的 V1 路径
git pull origin main

```

如果 `git pull` 报冲突：
```bash

git pull origin main
git stash pop         # 如果需要恢复
```

### Step 1.2 — 启动训练（20 分钟，含等待 learner 起来）

> **重要变更**：训练目录现在是**固定的** `outputs/train/paper_run_v1`（V1）/ `paper_run_v2`（V2）。
> 这样 Learner 和 Actor 写入同一目录，不会出现"两个时间戳分裂"的问题。
>
> 但代价是：**重跑前必须先归档旧目录**。直接用我们提供的 wrapper 脚本即可。

#### Step 1.2.1 — 跑 wrapper 脚本（10 秒）

```bash
cd ~/project/hilserl-surrol-improved/lerobot
./start_paper_run.sh
```

脚本会：
1. 如果 `outputs/train/paper_run_v1/` 已存在 → 自动重命名为 `paper_run_v1_archived_<时间戳>/`
2. 打印 Terminal A 和 Terminal B 该跑的命令

#### Step 1.2.2 — 跟随脚本提示

打开 **终端 A**（learner）—— 直接复制脚本提示中的命令：
```bash
cd ~/project/hilserl-surrol-improved/lerobot
conda activate hilserl
python -m lerobot.rl.learner --config_path train_config_gym_hil_touch.json
```

等待 learner 输出 `[LEARNER] Starting learner thread`（约 30 秒-1 分钟），再开 **终端 B**（actor）：
```bash
cd ~/project/hilserl-surrol-improved/lerobot
conda activate hilserl
python -m lerobot.rl.actor --config_path train_config_gym_hil_touch.json
```

> ⚠️ 如果跳过 `start_paper_run.sh` 直接启动 learner，且 `paper_run_v1/` 已存在，**会立即报 FileExistsError 退出**。这是设计上故意的，避免覆盖之前的数据。

### Step 1.3 — 5 分钟内必看的验证（关键！）

由于现在 output_dir 是固定路径，验证非常简单：

```bash
cd ~/project/hilserl-surrol-improved/lerobot/outputs/train/paper_run_v1/training_logs

# 1) 必须看到这 5 个文件
ls
# 期望：
#   training_metrics.csv          ← learner 训练日志（关键！）
#   episode_metrics.csv           ← episode 结束时的奖励
#   eval_metrics.csv              ← 暂时为空（无独立 eval）
#   experiment_metadata.json      ← 启动时立即生成
#   training_summary.json         ← checkpoint 时生成

# 2) 元数据立即就有
cat experiment_metadata.json | head -20
# 应看到 git_commit / git_branch / hostname / gpu / cfg_snapshot

# 3) 5 分钟后训练数据应该开始有了
wc -l training_metrics.csv      # 应 > 1（不只是表头）
wc -l episode_metrics.csv       # 应 ≥ 1
```

**如果 5 分钟后 `training_metrics.csv` 仍然只有表头（1 行）**，说明 learner 没在训练，立即按 [故障排查](#故障排查) 处理。

#### ⚠️ 历史教训（务必避免）

之前一次运行因为 `output_dir: null` 导致：
- Learner 启动时间戳 `11-48-55` → 写到 `outputs/train/2026-04-27/11-48-55_*/`
- Actor 启动时间戳 `11-49-17` → 写到 `outputs/train/2026-04-27/11-49-17_*/`

现在固定 `paper_run_v1` 后，**两个进程必然写到同一目录**，问题永久解决。

### Step 1.4 — 训练期间的干预策略（3 小时）

参考 [TOUCH_INTERVENTION_GUIDE.md](TOUCH_INTERVENTION_GUIDE.md)。本次实验的干预节奏建议：

| 训练时段 | 累计步数估计 | 干预频率 | 干预策略 |
|---------|-----------|---------|---------|
| 0-30 min  | 0-3K     | **80-100%**  | 几乎每个 episode 都引导抓起方块 |
| 30-90 min | 3K-15K   | **40-60%**   | 失败时立刻接管，成功时观察 |
| 90-180 min| 15K+     | **10-30%**   | 只在策略明显跑偏时才干预 |

**终端 B（actor）会实时打印**类似：
```
[ACTOR] Global step 1234: Episode reward: 1.0000 | intervention=YES (45%)
```
- `intervention=YES` 表示这个 episode 你干预过
- 百分比 = 该 episode 干预步数占比

2. 在 **终端 A（learner）** 按 `Ctrl+C`

## 故障排查

### 问题 0：启动时报 FileExistsError

**症状**：learner 终端立刻退出，错误信息为 `FileExistsError: Output directory outputs/train/paper_run_v1 already exists and resume is False`。

**原因**：你跳过了 `start_paper_run.sh`，直接启动 learner，但 `paper_run_v1/` 已存在（上次运行留下）。

**解决**：
```bash
cd ~/project/hilserl-surrol-improved/lerobot
./start_paper_run.sh   # 自动归档旧目录
# 然后重新跑 learner + actor
```

或手动归档：
```bash
mv outputs/train/paper_run_v1 outputs/train/paper_run_v1_old_$(date +%s)
```

### 问题 1：5 分钟后 `training_metrics.csv` 只有表头

**症状**：learner 没在训练。

**诊断**：
```bash
# 看 actor 是否在收集数据
tail -20 outputs/train/<...>/logs/actor_franka_sim_sac_touch.log
# 应看到 [ACTOR] Global step N

# 看 learner 是否还在 warmup
tail -20 outputs/train/<...>/logs/learner_franka_sim_sac_touch.log
# 如果一直在 [LEARNER] Warmup: X/50 ...，说明 actor 还没送够数据
```

**解决**：
- Warmup 阶段是正常的，等到累计 50 transitions 后会开始训练
- 如果 actor 长时间无 episode 结束（>2 分钟），可能是 Touch 设备没响应；重启 actor

### 问题 2：CUDA OOM

**症状**：learner 终端报 `CUDA out of memory`。

**解决**：
1. 编辑 `train_config_gym_hil_touch.json`，把 `batch_size: 128` 改为 `64`
2. 把 `online_buffer_capacity: 200000` 改为 `100000`
3. 重启 learner + actor
4. 在 `notes.txt` 中说明此事，论文需要记录

### 问题 3：Touch 设备无响应

**症状**：actor 启动后没有干预反应，按 Button 1 无效。

**解决**：
```bash
# 重新插拔 Touch 设备
# 然后重启 actor（不需要重启 learner）
# 在终端 B 按 Ctrl+C，等 actor 退出，再次执行 actor 启动命令
```

actor 重启后会自动重连 learner，已收集的训练数据不会丢。

### 问题 4：训练中途卡死（无新 episode）

**症状**：actor 终端长时间不输出新 `[ACTOR] Global step` 行。

**解决**：
1. 终端 B 按 Ctrl+C，等待 actor 干净退出（看到 `Flushing transitions`）
2. **不要** 关 learner
3. 再次启动 actor，会无缝续训

### 问题 5：V2 的 Q 值仍然很低（< 5）

**症状**：tail training_metrics.csv 看到 q_mean 一直在 0.5-2 之间，与 V2 设计的 15-40 不符。

**可能原因**：
- 不在 V2 仓库（确认路径含 `-v2`）
- V2 的 staged_reward_wrapper.py 没生效（检查 `cat lerobot/src/lerobot/rl/staged_reward_wrapper.py | head -5` 应看到 "Staged Dense Reward Wrapper v2"）

**解决**：
```bash
cd ~/project/hilserl-surrol-improved-v2
git log --oneline lerobot/src/lerobot/rl/staged_reward_wrapper.py | head -3
# 应包含 commit 7503535 "feat: redesign reward function for 1-2.5h convergence"
```
